# 🍔 قطة الدوام

**حل ذكي لمشكلة الطلبات الجماعية في بيئة العمل**

بوت تيليقرام يعيش في جروب الدوام، يفتح الطلب بعد `/start` في القروب، يستقبل المنيو بصيغة نصية عبر `/menu`, يسجّل طلب كل شخص، ثم يطلع ملخص الطلب النهائي داخل القروب فقط.

## 🎯 المشكلة اللي يحلها

> كل يوم نطلب أكل بالدوام، بس وقت الدفع ما نعرف مين طلب إيش — ونبلش نحسب يدوياً!

**قطة الدوام** يحل هذه المشكلة بالكامل:

| قبل | بعد |
|-----|-----|
| ❌ فوضى في الجروب | ✅ البوت ينظم كل شيء |
| ❌ ما تعرف مين طلب إيش | ✅ كل طلب مسجّل باسم صاحبه |
| ❌ حساب يدوي مزعج | ✅ البوت يحسب ويرسل لكل شخص نصيبه |
| ❌ إدخال المنيو يدوياً | ✅ يدخل المنيو مرة واحدة بصيغة نصية |

## 🚀 خطوات الإعداد

### الخطوة 1: إنشاء بوت تيليقرام

1. افتح تيليقرام وروح لـ **[@BotFather](https://t.me/BotFather)**
2. أرسل `/newbot`
3. اختار اسم للبوت مثلاً: `قطة دوامي`
4. اختار username مثلاً: `qattah_dawam_bot`
5. 📋 **احفظ الـ Token** — ستحتاجه لاحقاً

### الخطوة 2: الحصول على Claude API Key

1. روح على **[console.anthropic.com](https://console.anthropic.com)**
2. أنشئ حساباً أو سجّل دخول
3. من **API Keys** → أنشئ مفتاح جديد
4. 📋 **احفظ المفتاح**

> إذا ما عندك Claude API Key، البوت يظل يشتغل بالبدائل المحلية للخطوات الأساسية.

### الخطوة 3: تشغيل البوت

```bash
# 1. انسخ مجلد البوت
cd bot

# 2. أنشئ ملف الإعدادات
copy .env.example .env
```

عدّل ملف `.env`:
```env
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrSTUvwxYZ
CLAUDE_API_KEY=sk-ant-api03-...
```

```bash
# 3. شغّل البوت
start-bot.bat
```

ستشوف:
```
╔══════════════════════════════════════════╗
║      🤖 قطة الدوام - بوت تيليقرام        ║
╚══════════════════════════════════════════╝

  Bot: @qattah_dawam_bot
  API Server: http://192.168.1.5:3001

  ✅ البوت شغال! أضفه للجروب وابدأ.
```

### الخطوة 4: إضافة البوت للجروب

1. افتح جروب الدوام في تيليقرام
2. أضف البوت كعضو (ابحث عن `@username_bot`)
3. اعطه صلاحية إرسال الرسائل
4. ⚠️ **مهم:** اكتب `/start` داخل الجروب مرة واحدة لتفعيل البوت هناك

### كيف يشتغل البوت بعد التفعيل

1. تكتب `/start` داخل القروب، فيرد البوت بأنه صار مفعّل في هذا الجروب.
2. بعدها أي شخص يكتب عبارة مثل: `بنطلب من البيك` أو `نطلب من بيت الشاورما`.
3. البوت يحدد اسم المطعم ويفتح جلسة طلب جديدة.
4. ترسل المنيو مرة واحدة باستخدام `/menu` بصيغة نصية بسيطة.
5. كل شخص يكتب طلبه بالعربي، والبوت يحوله إلى أصناف من المنيو مع السعر.
6. عند إرسال الطلب، البوت يخزن طلب كل شخص مع اسمه والسعر.
7. عند كتابة `تم الطلب` أو `/done`، البوت يقفل الجلسة ويعرض ملخص نهائي في القروب.

### صيغة المنيو

أفضل صيغة هي:

```text
/menu
برجر:
- دجاج كريسبي - 25
- بيق تشيكن - 30
مشروبات:
- بيبسي - 5
```

يمكنك أيضاً استخدام `:` أو `|` أو كتابة السعر بعد اسم الصنف مباشرة.

### الخطوة 5: تطبيق الجوال (اختياري)

```bash
cd QattahMobile
npm install
npx expo start
```

اسكان QR Code من تطبيق **[Expo Go](https://expo.dev/go)** على جوالك، ثم في شاشة الإعدادات أدخل:
```
http://192.168.1.5:3001   ← رابط السيرفر اللي ظهر عند تشغيل البوت
```

## 📖 طريقة الاستخدام اليومية

### في جروب تيليقرام:

**1️⃣ بدء الطلب**
```
أنت: /start
البوت: ✅ تم تفعيل البوت في القروب.

أنت: بنطلب من البيك اليوم
```
```
البوت: 🍔 رائع! طلب من البيك
        أرسل المنيو الآن باستخدام /menu
```

**2️⃣ إرسال المنيو**

```text
/menu
برجر:
- دجاج كريسبي - 25
- بيق تشيكن - 30
```

البوت يحفظ المنيو ويبدأ يستقبل الطلبات.

**3️⃣ الكل يطلب بالعربي**
```
أحمد:   أبغى كريسبي مع كول سلو
البوت:  ✅ @أحمد: دجاج كريسبي × 1 + كول سلو × 1 = 33 ريال — صح؟
        [ ✅ أيوه صح ]  [ ❌ لا، عدّل ]

محمد:   واحد بيق تشيكن
البوت:  ✅ @محمد: بيق تشيكن × 1 = 30 ريال — صح؟
```

**4️⃣ إغلاق الطلب**
```
أنت:    تم الطلب
البوت:  ✅ تم إغلاق الطلب!
```

**5️⃣ الملخص النهائي في الجروب**
```
📊 ملخص الطلب — البيك

• أحمد: دجاج كريسبي × 1 + كول سلو × 1 = 33.00 ريال
• محمد: بيق تشيكن × 1 = 30.00 ريال

💰 الإجمالي: 63.00 ريال
👥 2 أشخاص
```

## 🤖 أوامر البوت

| الأمر | الوظيفة | أين يُستخدم |
|-------|---------|------------|
| `/start` | تفعيل البوت داخل الجروب | الجروب 👥 |
| `/menu` | إدخال المنيو النصي | الجروب 👥 |
| `/order [مطعم]` | بدء طلب جديد | الجروب 👥 |
| `/status` | عرض الطلبات الحالية | الجروب 👥 |
| `/done` | إغلاق الطلب وعرض الملخص | الجروب 👥 |

### تفاصيل الأوامر

- `/start`: يفعّل البوت داخل القروب فقط، وبعدها يبدأ يستقبل عبارات الطلب.
- `/menu`: يحفظ المنيو النصي داخل الجلسة الحالية ويجهز البوت لاستقبال الطلبات.
- `/order [مطعم]`: اختصار لبدء طلب جديد مباشرة بدون كتابة صيغة `بنطلب من`.
- `/status`: يعرض الطلب المفتوح حالياً وعدد الطلبات المؤكدة والإجمالي.
- `/done`: يقفل الطلب الحالي ويطبع الملخص النهائي داخل القروب.

## ⚠️ ملاحظات مهمة

> **المنيو:** النسخة الحالية تعتمد على إدخال المنيو النصي داخل القروب عبر `/menu` بدل سحب البيانات من تطبيقات التوصيل.

> **النسخة الحالية:** الملخص النهائي يظهر داخل الجروب فقط، ولا توجد رسائل خاصة أو إعدادات دفع.

> **الشبكة:** التطبيق الجوال يتصل بالسيرفر عبر الشبكة المحلية. إذا أردت وصولاً من أي مكان، يمكن نشر البوت على سيرفر cloud مثل Railway أو Render.
